# Building a Custom ComfyUI Node Web UI & Companion API

This guide documents the architecture and implementation patterns used to build the Smart Prompt Generator floating UI panel and its Python API layer. It is written to be general so you can adapt the same approach for ANY custom node or tool you add to ComfyUI.

---
## 1. Goals & Design Principles

**Objectives**
- Expose node functionality over HTTP endpoints (REST‑like) for automation and external tools.
- Provide an in‑browser floating panel (no bundler required) for interactive exploration.
- Keep the solution self‑contained inside the custom node folder (easy install / remove).
- Fail gracefully: UI still loads even if some endpoints are missing; generation errors surface with context.
- Offer extensibility: settings, batch processing, health diagnostics, optional emphasis or formatting.

**Principles**
- *Progressive enhancement*: Simple JSON endpoints first, then UI features layered incrementally.
- *Loose coupling*: Node business logic stays in Python classes; API layer is a thin wrapper; UI only speaks HTTP.
- *Stateless endpoints*: Each request carries what it needs; only small shared state (stats, batch jobs) in memory / yaml.
- *Minimal dependencies*: `aiohttp` only on Python side; vanilla JS + inline `<style>` on frontend.
- *Discoverability*: Health endpoint, version endpoint, structured error messages with a `stage` field.

## 3. Bootstrapping API Routes

In `__init__.py` we detect the ComfyUI server and register routes:

```python
# __init__.py
from .nodes import NODE_CLASS_MAPPINGS, NODE_DISPLAY_NAME_MAPPINGS
WEB_DIRECTORY = "./web"

try:
    from . import api
    import server
    api.setup_api_routes(server.PromptServer.instance.app)
except ImportError:
    pass  # Server not available (e.g., offline tooling)
except Exception as e:
    print(f"Warning: Could not setup API routes: {e}")
```

Key idea: separation of *node* logic and *web* logic; the API can fail without breaking node registration.

## 4. API Layer (aiohttp)

### 4.1 Core Patterns
- Define a module‑level singleton of your node engine (`_GEN = YourNode()`).
- Build `@routes.get` / `@routes.post` handlers that:
  1. Parse & validate input.
  2. Invoke engine methods.
  3. Return JSON with `success: True/False`.
- Wrap errors: return 4xx for validation, 5xx for internal failures.
- Add a `/health` endpoint to surface environment or dependency issues.

### 4.2 Example Endpoint Skeleton
```python
from aiohttp import web
routes = web.RouteTableDef()

@routes.post('/your/generate')
async def generate_endpoint(request: web.Request):
    body = await safe_json(request)
    try:
        param = body.get('param', 'default')
        output = ENGINE.generate(param)
        return web.json_response({"success": True, "result": output})
    except ValueError as ve:
        return web.json_response({"success": False, "error": str(ve)}, status=400)
    except Exception as e:
        return web.json_response({"success": False, "error": f"internal_error: {e}"}, status=500)
```

### 4.3 Safe JSON Parsing
Older `aiohttp` versions can be strict about `Content-Type`. Implement a tolerant helper:
```python
def json_or_empty(text: str):
    if not text: return {}
    import json as _j
    try: return _j.loads(text)
    except Exception: return {}

async def safe_json(request):
    try:
        return await request.json()
    except Exception:
        return json_or_empty(await request.text())
```

### 4.4 Stage‑Aware Errors (Optional)
Wrap internal pipeline stages to aid debugging:
```python
try:
    stage = "load"
    data = load_data()
    stage = "transform"
    result = transform(data)
    stage = "render"
    return render(result)
except Exception as e:
    raise RuntimeError(f"stage:{stage}:{e}") from e
```
API handler then extracts `stage` and returns it in JSON for UI display.

### 4.5 Health & Version Endpoints
```python
@routes.get('/your/health')
async def health(_):
    ok, err = True, None
    try: ENGINE.self_test()
    except Exception as e: ok, err = False, str(e)
    return web.json_response({"ok": ok, "error": err})

@routes.get('/api/version')  # compatibility for scripts
async def version(_):
    return web.json_response({"success": True, "version": "1.0.0"})
```

### 4.6 Batch / Async Jobs (Pattern)
- Maintain a dict: `{ job_id: {status, progress, results, cancel_flag} }`.
- Use `threading.Thread` or `asyncio.create_task` for long operations.
- Poll with `/batch_status` and retrieve with `/batch_results`.

Minimal example:
```python
import threading, time, uuid
BATCH = {}

@routes.post('/your/batch_start')
async def batch_start(request):
    job_id = uuid.uuid4().hex
    cancel = threading.Event()
    BATCH[job_id] = {"status": "running", "progress": 0, "results": [], "cancel": cancel}
    threading.Thread(target=_run, args=(job_id,), daemon=True).start()
    return web.json_response({"success": True, "job_id": job_id})

def _run(job_id):
    job = BATCH[job_id]
    for i in range(10):
        if job['cancel'].is_set():
            job['status'] = 'cancelled'; return
        time.sleep(0.2)
        job['progress'] = (i+1)*10
        job['results'].append(f"item_{i}")
    job['status'] = 'completed'
```

## 5. Frontend Floating Panel (Vanilla JS)

### 5.1 Injection Strategy
Placing JavaScript in `web/js/your_ui.js` automatically serves it. The script immediately creates a panel once DOM is ready, unless gated by a setting.

### 5.2 Panel Structure
- Container div with header (drag handle + buttons).
- Several `<section>` blocks for search, generation, analysis, batch, stats.
- Inline `<style>` appended programmatically to avoid external CSS file conflicts.

### 5.3 Core Helpers
```javascript
async function apiGet(path, params={}) {
  const url = new URL('/your'+path, location.origin);
  Object.entries(params).forEach(([k,v])=>{ if(v!=null && v!=='') url.searchParams.set(k,v); });
  const r = await fetch(url); if(!r.ok) throw new Error(r.status); return r.json();
}
async function apiPost(path, body={}) {
  const r = await fetch('/your'+path,{method:'POST',headers:{'Content-Type':'application/json'},body:JSON.stringify(body)});
  let json; try { json = await r.json(); } catch { json = {}; }
  if(!r.ok || json.success===false) throw new Error(json.error || ('HTTP '+r.status));
  return json;
}
```

### 5.4 Error Surface with Stage
```javascript
try { const res = await apiPost('/generate', payload); }
catch(e){ outputEl.textContent = 'Error: '+e.message; }
```
If backend returns `{ success:false, error:"...", stage:"formatting" }` you can append `(stage formatting)` for clarity.

### 5.5 Search Lists
Render simple lists; clicking injects chosen value into selector:

```javascript
function renderList(el, items, onPick){

```

### 5.6 Drag & Persist Position (Settings Integration)
```javascript
let dragging=false, offset={};
header.onmousedown = e=>{ dragging=true; offset.x=e.offsetX; offset.y=e.offsetY; };
```

Expose methods so other extensions or settings toggles can manipulate the panel:

## 9. Batch & Async Patterns
**When to use**: expensive or high‑volume operations (prompt sets, multi‑style comparisons).

**Workflow**:
1. Client POST `/batch_start` with parameters.
2. Server spawns thread or asyncio task; returns `batch_id`.
3. Client polls `/batch_status?batch_id=...` until `status` is `completed|error|cancelled`.
4. Client fetches final data via `/batch_results`.
5. Optional cancellation with `/batch_cancel`.

**Client Polling Snippet**:
```javascript
let pollTimer;
async function start(){
  const {batch_id} = await apiPost('/batch_start', cfg);
  pollTimer = setInterval(async()=>{
    const s = await apiGet('/batch_status',{batch_id});
    if(s.data.status !== 'running'){ clearInterval(pollTimer); const r = await apiGet('/batch_results',{batch_id}); console.log(r); }
  }, 1500);
}
```

## 11. Security & Stability Notes
- Avoid writing untrusted text directly to files—sanitize or restrict APIs.
- Keep shared mutable globals minimal (e.g., stats lock, batch registry).
- Clamp or validate numeric inputs (word count sizes, batch lengths) to prevent resource abuse.
- Wrap every non‑trivial endpoint body in `try/except` and return consistent JSON shape.

## 14. Adapting to Your Node
1. Identify core method(s) you want remote & UI access to (e.g., `generate`, `analyze`, `transform`).
2. Wrap them in safe endpoints with validation.
3. Build incremental UI: start with a single button & output `<pre>`, then expand.
4. Introduce settings only after core flow is stable.
5. Add batch/asynchronous features when you have a clear need for multiple parallel or sequential operations.

## 16. Troubleshooting Quick Reference
| Symptom | Likely Cause | Fix |
|---------|--------------|-----|
| 404 on endpoint | Routes not registered | Ensure `setup_api_routes` called in `__init__` after server import |
| 500 with `stage:xyz` | Internal stage failed | Inspect stack; add logging around that stage |
| UI panel not showing | Setting disabled or script load order | Check `auto_show_panel`; call `window.smartPromptOpenPanel()` |
| Batch stuck running | Thread error swallowed | Add logs inside loop; surface job['error'] |
| Stats not updating | File permission or exception | Wrap writes in `try/except` & log once |

### 18.2 Safe Stats Write (Atomic-ish)

If you update stats, a crash mid-write can corrupt the file. Easiest mitigation:

```python
import json, os, tempfile
def atomic_write(path, data):
    d = os.path.dirname(path)
    fd, tmp = tempfile.mkstemp(dir=d, prefix='.tmp_stats_')
    try:
        with os.fdopen(fd, 'w') as f: json.dump(data, f, indent=2)
        os.replace(tmp, path)  # atomic on POSIX
    except Exception:
        try: os.remove(tmp)
        except OSError: pass
```

Call this instead of directly opening the stats file. Good enough; no heavy locking unless you have multiple writer threads (you likely don't).
